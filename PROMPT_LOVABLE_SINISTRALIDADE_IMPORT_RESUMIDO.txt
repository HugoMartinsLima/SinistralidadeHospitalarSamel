APIS DE SINISTRALIDADE IMPORTADA - CORREÇÃO FINAL

As APIs foram corrigidas para:
1. Usar colunas em camelCase com aspas duplas (padrão Oracle)
2. Filtrar por "dtProcedimento" (data do procedimento) ao invés de "data"

=== ENDPOINT: GET /api/sinistralidade/contratos/resumo ===

Query params:
- dataInicio (obrigatório): DD/MM/YYYY ou DD/MM/YY (ex: 01/11/2025 ou 01/11/25)
- dataFim (obrigatório): DD/MM/YYYY ou DD/MM/YY
- contratos (opcional): lista separada por vírgula (ex: 2444,2445)
- grupoReceita (opcional): filtro por grupo

Exemplo:
GET /api/sinistralidade/contratos/resumo?dataInicio=01/11/2025&dataFim=30/11/2025&contratos=2444

Resposta:
{
  "data": [{
    "apolice": 2444,
    "contratante": "2E DESPACHOS ADUANEIROS LTDA",
    "sinistroTotal": 15000.50,
    "sinistroTitular": 10000.00,
    "sinistrosDependentes": 5000.50,
    "quantidadeBeneficiarios": 25,
    "quantidadeAtendimentos": 150,
    "breakeven": 75
  }],
  "total": 1
}

=== ENDPOINT: GET /api/sinistralidade/detalhamento ===

Query params:
- nrContrato (obrigatório): número da apólice
- dataInicio (obrigatório): DD/MM/YYYY ou DD/MM/YY
- dataFim (obrigatório): DD/MM/YYYY ou DD/MM/YY
- grupoReceita (opcional): filtro por grupo
- limit (opcional): paginação
- offset (opcional): paginação

Exemplo:
GET /api/sinistralidade/detalhamento?nrContrato=2444&dataInicio=01/11/2025&dataFim=30/11/2025

Resposta:
{
  "data": [{
    "data": "15/11/2025",
    "hora": "14:30",
    "dtProcedimento": "15/11/2025",
    "apolice": 2444,
    "contratante": "2E DESPACHOS ADUANEIROS LTDA",
    "beneficiario": "JOAO SILVA",
    "valorTotal": 150.00,
    "grupoReceita": "CONSULTAS",
    ... (45 campos no total)
  }],
  "total": 150
}

=== ENDPOINT: GET /api/sinistralidade/grupos-receita ===

Sem parâmetros.

Exemplo:
GET /api/sinistralidade/grupos-receita

Resposta:
{
  "data": [
    { "grupoReceita": "CONSULTAS" },
    { "grupoReceita": "EXAMES" },
    { "grupoReceita": "INTERNACAO" }
  ],
  "total": 3
}

=== ENDPOINT: GET /api/sinistralidade/pacientes/busca ===

Query params:
- nome (obrigatório, min 3 chars): nome do paciente
- dataInicio (obrigatório): DD/MM/YYYY ou DD/MM/YY
- dataFim (obrigatório): DD/MM/YYYY ou DD/MM/YY
- grupoReceita (opcional): filtro por grupo

Exemplo:
GET /api/sinistralidade/pacientes/busca?nome=JOAO&dataInicio=01/11/2025&dataFim=30/11/2025

Resposta:
{
  "data": [...],
  "total": 25,
  "paciente": "JOAO"
}

=== IMPORTANTE ===

- Todas as colunas usam camelCase (beneficiario, valorTotal, dtProcedimento, etc)
- O filtro de período usa a coluna "dtProcedimento" (data do procedimento)
- Datas podem ser enviadas como DD/MM/YYYY ou DD/MM/YY
- A API não usa path parameters, apenas query parameters
