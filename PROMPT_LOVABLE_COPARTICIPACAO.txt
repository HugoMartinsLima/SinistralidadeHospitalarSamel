## Alteração: Novo campo vlCoparticipacao no endpoint GET /api/sinistralidade/contratos/resumo

### Descrição
O endpoint agora retorna um novo campo `vlCoparticipacao` que representa o valor de coparticipação mensal do contrato.

### Endpoint Afetado
GET /api/sinistralidade/contratos/resumo

### Novo Campo na Resposta

| Campo | Tipo | Descrição |
|-------|------|-----------|
| vlCoparticipacao | number | Valor total de coparticipação do período (vl_mensalidade_copart) |

### Exemplo de Resposta Atualizada
```json
{
  "data": [
    {
      "apolice": 263,
      "contratante": "EMPRESA EXEMPLO LTDA",
      "sinistroTotal": 125450.80,
      "sinistroTitular": 98200.50,
      "sinistrosDependentes": 27250.30,

      "quantidadeAtendimentos": 312,
      "breakeven": 75,

      "vlCoparticipacao": 3500.00  // NOVO CAMPO

  "total": 1
}
```

### Impacto
- Nenhuma alteração nos parâmetros de requisição
- Campo retornado com valor 0 se não houver dados de coparticipação
- Compatível com versões anteriores (campo adicional)
